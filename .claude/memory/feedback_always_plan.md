ALWAYS create a plan in the Hive plan system before starting implementation work. Research first, plan second, build third.

**Why:** Kyle corrected me for jumping straight to code on the Hive Channel feature. The plan system exists for a reason — it ensures alignment, tracks progress, and gives visibility to all agents.

**How to apply:**
1. Research the problem — read docs, check existing code, understand constraints
2. Create a plan via `hive_plan_create` with checklist items
3. Get alignment from Kyle if the scope is non-trivial
4. THEN start building, checking off items as you go
5. Never skip straight to writing code on new features
